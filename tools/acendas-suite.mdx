The Acendas Atlassian Suite is an open-source TypeScript MCP plugin for Claude Code that exposes 179 tools across JIRA, Confluence, and Bitbucket, plus 29 cross-product skills and 7 orchestrator agents for automated PR reviews, sprint retros, and release notes. Install it once and Claude Code gains deep, native awareness of your entire Atlassian ecosystem — no custom prompting, no fragile shell scripts.

## Coverage

The suite organises its 179 tools into four product areas, each with its own set of discrete actions:

- **JIRA**: Issue CRUD, sprint management, work breakdown, label and priority handling
- **Confluence**: Page creation, architecture decision records (ADRs), living runbooks, space search
- **Bitbucket**: PR creation, review, approval, branch management, pipeline status
- **Cross-product skills**: `standup-notes`, `sprint-retro`, `release-notes`, `review-pr`, `link-pr-to-issue` — composite workflows that orchestrate multiple product areas in a single command

## 8-Agent PR Review Pipeline

The suite's most powerful feature is its three-wave PR review pipeline. Give Claude Code the `review-pr` skill and it spawns eight specialised sub-agents automatically:

- **Wave 1 — Parallel scanners (6 agents)**: Security, performance, style, logic, test coverage, and documentation scanners run simultaneously, each focused on its own domain.
- **Wave 2 — Investigator**: A synthesis agent reads all six scanner reports, deduplicates overlapping findings, and produces a unified review with ranked recommendations.
- **Wave 3 — Adversarial critic**: A final agent challenges the investigator's proposed review, surfacing blind spots and false positives before the review is posted to Bitbucket.

The result lands as a structured Bitbucket PR comment with severity labels and line-level annotations.

## Installation

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/acendas/atlassian-suite
    cd atlassian-suite
    ```
  </Step>
  <Step title="Install dependencies">
    ```bash
    npm install
    ```
  </Step>
  <Step title="Configure credentials">
    Copy `.env.example` to `.env` and fill in your API tokens:

    ```bash .env
    JIRA_URL=https://yourorg.atlassian.net
    JIRA_TOKEN=your-jira-api-token
    JIRA_EMAIL=you@yourorg.com

    CONFLUENCE_URL=https://yourorg.atlassian.net/wiki
    CONFLUENCE_TOKEN=your-confluence-api-token

    BITBUCKET_URL=https://api.bitbucket.org/2.0
    BITBUCKET_TOKEN=your-bitbucket-app-password
    ```
  </Step>
  <Step title="Register the MCP server in Claude Code">
    Add the following entry to your Claude Code MCP settings (`.claude/mcp.json` or your IDE's MCP config panel):

    ```json .claude/mcp.json
    {
      "mcpServers": {
        "acendas-atlassian": {
          "command": "node",
          "args": ["./atlassian-suite/dist/server.js"]
        }
      }
    }
    ```

    Restart Claude Code to load the 179 tools.
  </Step>
</Steps>

## Sample Skills

Once installed, use natural language prompts directly in Claude Code. The suite maps these to the correct sequence of API calls automatically:

- `"Run standup-notes for team alpha sprint 14"` — pulls completed, in-progress, and blocked issues and formats a standup summary
- `"Generate release notes for version 2.3.0"` — scans merged PRs and resolved JIRA issues since the last tag and writes a structured changelog
- `"Review PR #456 using the full 8-agent pipeline"` — triggers the three-wave review pipeline and posts findings to Bitbucket
- `"Link PR #456 to JIRA issue PROJ-789"` — updates the PR description and the JIRA issue with mutual cross-links

<Note>
The Acendas Atlassian Suite is designed for **Atlassian Cloud + Bitbucket**. If your team uses GitLab or Gitea for version control, use the [JIRA GitLab Helper](/tools/jira-gitlab-helper) instead — it provides the same JIRA-to-MR automation against a GitLab backend.
</Note>
