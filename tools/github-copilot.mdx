GitHub Copilot integrates with JIRA to let you select an issue, choose an AI model (GPT, Claude, or Gemini), review a generated implementation plan, and trigger code generation that opens a linked pull request automatically. The integration tightens the loop between issue tracking and the GitHub repository, reducing the manual steps between a ticket being assigned and a branch appearing in your codebase.

## Key Features

**GitHub Copilot for Jira** is the official first-party integration between GitHub Copilot and Atlassian JIRA Cloud. Once installed, a Copilot icon appears inside every JIRA issue. You select the issue, pick your preferred AI model from a dropdown (GPT-4o, Claude Sonnet, or Gemini Pro), review the plan Copilot generates from the issue description and acceptance criteria, and then approve it. Copilot creates the branch in your connected GitHub repository and opens a draft PR with the issue key in the title — automatically. Teams using this integration report roughly a **50% reduction** in issue-to-PR time compared to manual workflows.

**Rovo for GitHub Copilot** is an extension developed by Atlassian that adds an `@atlassian-rovo` participant to the Copilot chat panel in VS Code and other supported IDEs. With it installed, you can pull live JIRA issue data and Confluence documentation directly into your Copilot conversation without switching to a browser tab, keeping your full project context within arm's reach while you code.

**Copilot Workspace** is GitHub's multi-step autonomous agent environment, available at [copilot-workspace.githubnext.com](https://copilot-workspace.githubnext.com). It takes any GitHub issue or natural-language specification, builds a detailed spec, produces an implementation plan, writes the code across multiple files, and opens a PR for review. For complex cross-cutting changes — refactors, feature flags, or API integrations — Copilot Workspace handles the coordination that would otherwise require many manual steps.

## Setting Up the JIRA Integration

<Steps>
  <Step title="Install GitHub Copilot">
    You need an active GitHub Copilot subscription (Individual, Business, or Enterprise). Confirm your subscription is active at [github.com/settings/copilot](https://github.com/settings/copilot). Install the GitHub Copilot extension in your IDE (VS Code, IntelliJ IDEA, or JetBrains suite) and sign in with your GitHub account.
  </Step>
  <Step title="Install the GitHub Copilot app in JIRA Cloud">
    In your JIRA Cloud instance, navigate to **Settings → Apps → Find new apps** (Atlassian Marketplace). Search for **GitHub Copilot** and install the official app published by GitHub. You need JIRA administrator permissions to install Marketplace apps.
  </Step>
  <Step title="Authorize GitHub OAuth">
    After installation, open the app configuration page at **Settings → Apps → GitHub Copilot → Configure**. Click **Authorize with GitHub** and complete the OAuth flow. Grant the app access to the GitHub organization and repositories you want to link to JIRA projects.
  </Step>
  <Step title="Select your AI model from a JIRA issue">
    Open any JIRA issue. You will see a new **Copilot** button in the issue toolbar (top-right area). Click it to open the Copilot panel. From the **Model** dropdown, select your preferred AI model — GPT-4o, Claude 3.5 Sonnet, or Gemini Pro.
  </Step>
  <Step title="Review the plan and create the PR">
    Copilot reads the issue summary, description, and acceptance criteria, then generates a step-by-step implementation plan. Review the plan in the panel. If the plan looks correct, click **Approve and Generate**. Copilot creates a new branch named `copilot/<issue-key>-<slug>` in your repository and opens a draft pull request linked to the JIRA issue. The PR description includes the full plan and a link back to the ticket.
  </Step>
</Steps>

## Using @atlassian-rovo in Copilot Chat

The Rovo extension for GitHub Copilot brings JIRA and Confluence context into the Copilot chat panel, so you can ask questions about your project without leaving the IDE.

<Steps>
  <Step title="Install the Rovo extension for Copilot">
    In VS Code, open the Extensions panel and search for **Atlassian Rovo**. Install the extension and sign in to your Atlassian account when prompted. The extension registers `@atlassian-rovo` as a chat participant in the GitHub Copilot Chat panel.
  </Step>
  <Step title="Query JIRA and Confluence from chat">
    Open the Copilot Chat panel (`Ctrl+Alt+I` / `Cmd+Option+I`). Type `@atlassian-rovo` followed by your query. For example:

    ```text
    @atlassian-rovo What are the acceptance criteria for PROJ-789?
    ```

    ```text
    @atlassian-rovo Summarize the architecture decisions in the "Auth Service" Confluence space.
    ```

    ```text
    @atlassian-rovo List all open bugs in the PAYMENTS project assigned to me.
    ```

    The extension fetches live data from your Atlassian Cloud instance and injects it into the conversation context, allowing Copilot to use it when generating code or explanations.
  </Step>
</Steps>

## Compatibility Note

The GitHub Copilot for JIRA integration and the automatic PR creation workflow require **GitHub.com** (cloud-hosted repositories). The OAuth connection between JIRA Cloud and your repository is established through GitHub's cloud API.

If your team uses a self-hosted Git platform — such as Gitea, GitLab CE, or Bitbucket Data Center — you can still use GitHub Copilot inside your IDE for AI-assisted code completion and chat. However, the automatic branch creation and PR opening features will not be available, as they depend on the GitHub.com repository API. In those environments, use Copilot for in-IDE code generation and create PRs manually, or explore the Atlassian Rovo MCP Server as an alternative agent integration layer.

<Tip>
  For complex multi-file changes — such as introducing a new API layer, refactoring a data model, or adding feature flags across a service — use **Copilot Workspace** instead of the JIRA issue panel. Copilot Workspace gives you full visibility into the spec, plan, and diff at every step, and lets you iterate on the plan before any code is written. Start a Workspace session from any GitHub issue at [copilot-workspace.githubnext.com](https://copilot-workspace.githubnext.com).
</Tip>
